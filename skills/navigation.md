# Navigation: skills/

## What lives here?

All active skills. Each skill is a sub-directory with `SKILL.md` as
SoT. Optional further files: `REFERENCE.md` for off-loaded detail,
prompts / templates / examples — cross-reference rather than
duplicate.

**Skill format:** v2 is mandatory. Frontmatter with `description`
(use-when), `invocation.primary`. Format
standard: `framework/skill-anatomy.md`. Validator:
`scripts/skill_fm_validate.py` (pre-commit check 7, BLOCK).

**Classification:** single-class model. All skills are equal.
Workflow is a separate layer (`workflows/runbooks/`); protocols are
a separate asset type (`_protocols/`).

**`_protocols/`:** shared loads, referenced by other skills, never
called directly. Its own navigation lives in
`_protocols/navigation.md`.
